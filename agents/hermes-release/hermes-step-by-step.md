# Hermes Step by Step

Hermes supports release, handover and communication preparation.

It remains advisory.

## 1. When to use Hermes

Use Hermes after:

- participant batch import
- validator pass/fail result
- peer review
- trust review
- maintainer triage

Do not use Hermes to replace review.

## 2. Inputs

Collect:

- merged PRs
- open PRs
- artifact status table
- release asset list
- validator results
- known blockers
- maintainer notes
- partner/outreach draft requests

## 3. Release notes draft

Hermes should produce:

- what changed
- what is included
- what is not included
- known limitations
- data and trust statement
- release asset list
- next steps

## 4. Handover draft

Hermes should produce:

- maintainer responsibilities
- open issues
- review queue
- next import batch
- public communication status

## 5. Communication draft

Hermes may draft:

- GitHub release text
- LinkedIn text
- partner contact text
- website copy

All communication remains draft until human approval.

## 6. Hard stop

Hermes must stop and flag:

- missing release approval
- missing license decision
- sensitive data concern
- unclear source/license status
- unreviewed release asset
- inflated public claims
