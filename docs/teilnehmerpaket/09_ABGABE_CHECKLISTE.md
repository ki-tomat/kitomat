# 09 - Abgabe-Checkliste

## Allgemein

- [ ] Ich habe genau ein Hauptartefakt.
- [ ] Der Artefakttyp ist klar.
- [ ] Zielgruppe ist beschrieben.
- [ ] Use Case ist konkret.
- [ ] Status ist `draft` oder `bronze_candidate`.
- [ ] Keine Platzhalter bleiben offen.

## Pflichtdateien

- [ ] Alle Pflichtdateien fuer meinen Artefakttyp sind vorhanden.
- [ ] `metadata.yml` ist ausgefuellt.
- [ ] README ist verstaendlich.
- [ ] Beispiele sind vorhanden.
- [ ] Failure Modes oder Grenzen sind beschrieben.

## Szenario-Triade

- [ ] positives Szenario vorhanden.
- [ ] nachbearbeitbares Szenario vorhanden.
- [ ] negatives Szenario vorhanden.
- [ ] Expertenfeedback zu jedem Szenario vorhanden.

## Daten und Quellen

- [ ] Keine echten personenbezogenen Daten.
- [ ] Keine Kundendaten.
- [ ] Keine internen Dokumente.
- [ ] Keine Gesundheitsdaten.
- [ ] Keine vertraulichen Finanzdaten.
- [ ] Keine unklaren Volltexte.
- [ ] Quellen sind dokumentiert.
- [ ] Lizenzstatus ist dokumentiert.
- [ ] Abrufdatum ist dokumentiert, wenn Quellen genutzt wurden.

## Downloads

Falls groessere Dateien genutzt werden:

- [ ] Datei nicht direkt ins Git-Repo gelegt.
- [ ] Release-Asset-Beschreibung vorhanden.
- [ ] Version vorhanden.
- [ ] Groesse vorhanden oder als offen markiert.
- [ ] SHA-256-Pruefsumme vorhanden oder als offen markiert.
- [ ] Datenrisiko dokumentiert.

## Review

- [ ] Offene Fragen sind klar markiert.
- [ ] Peer Review ist moeglich.
- [ ] Trust Review ist angefordert, wenn Risiko `yellow` oder `red`.
- [ ] Ich behaupte keine rechtliche, medizinische, HR- oder Finanzfreigabe.
