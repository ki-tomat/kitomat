# 02 - Teilnehmer Step by Step

## Ziel

Du erstellst in 6 bis 9 Stunden ein hochwertiges Hauptartefakt fuer KItomat. Du darfst KI-Tools nutzen, aber dein Fachwissen entscheidet.

## Schritt 1 - Hauptartefakt waehlen

Waehle genau eines:

- Prompt-Paket
- Datensatz-/Quellenpaket
- KMU-/Branchenmodell

Beginne kein zweites Hauptartefakt, bevor dein erstes mindestens `bronze_candidate`-faehig ist.

## Schritt 2 - Praxisproblem beschreiben

Beantworte kurz:

- Fuer wen ist das Artefakt?
- Welches konkrete Problem loest es?
- Welche Eingaben braucht es?
- Was soll am Ende herauskommen?
- Wo liegen Risiken oder Grenzen?

## Schritt 3 - Mit KI eine Rohstruktur erstellen

Nutze Claude, ChatGPT oder ein anderes Tool, um eine erste Struktur zu erzeugen.

Wichtig: Die KI darf strukturieren, aber sie darf dein Fachwissen nicht ersetzen.

## Schritt 4 - Pflichtdateien ausfuellen

Nutze den passenden Artefakttyp aus `03_ARTEFAKT_TYPEN_UND_PFLICHTDATEIEN.md`.

Fuellen:

- README
- metadata.yml
- Beispiele
- Evaluation oder Anwendungshinweise
- Failure Modes
- Quellen oder Quellenstatus

## Schritt 5 - Szenario-Triade erstellen

Erstelle:

1. positives Szenario
2. nachbearbeitbares Szenario
3. negatives Szenario

Schreibe dazu jeweils ein kurzes Expertenfeedback.

## Schritt 6 - Daten und Quellen pruefen

Erlaubt:

- synthetische Beispieldaten
- eigene Musterfaelle ohne echte Personen
- Quellenlisten
- kurze eigene Zusammenfassungen mit Quellenangabe
- oeffentliche Quellen mit Abrufdatum

Nicht erlaubt:

- echte Kundendaten
- Gesundheitsdaten
- interne Dokumente
- personenbezogene E-Mails
- vertrauliche Finanzdaten
- unklare Volltexte

## Schritt 7 - Abgabeweg waehlen

Weg A: GitHub direkt

1. Fork oder Branch nutzen.
2. Dateien in die passende Struktur legen.
3. Pull Request erstellen.
4. Review-Fragen beantworten.

Weg B: Import durch Dozent/Maintainer

1. Dateien lokal vorbereiten.
2. An Dozent/Maintainer weitergeben.
3. Codex oder Maintainer importiert in einen Branch.
4. Du beantwortest Review-Fragen.

## Schritt 8 - Review

Dein Beitrag wird geprueft auf:

- Vollstaendigkeit
- Verstaendlichkeit
- Nutzbarkeit
- Beispielqualitaet
- Szenario-Triade
- Quellen
- Datenrisiko
- Grenzen und Failure Modes

Ziel ist `bronze_candidate`. Finales `bronze` entscheidet nur ein Maintainer nach Review.
