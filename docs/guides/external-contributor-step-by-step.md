# External Contributor Step by Step

This guide is for people outside the course who want to contribute.

## 1. Read first

Read:

- `README.md`
- `CONTRIBUTING.md`
- `docs/community-contribution-flow.md`
- the matching artifact template

## 2. Use fork plus pull request

1. Fork `ki-tomat/kitomat`.
2. Create a branch in your fork.
3. Copy one artifact template.
4. Fill the required files.
5. Open a pull request to `ki-tomat/kitomat:main`.

Do not expect direct push access.

## 3. Contribution rules

- one main artifact per pull request
- no real personal data
- no customer data
- no internal company documents
- no unclear full-text copies
- no legal, audit, medical, HR or finance decision claims
- no final status above `bronze_candidate`

## 4. Review path

Your PR will go through:

1. automated validators
2. OpenClaw/Codex advisory precheck if used
3. peer review when needed
4. trust review when needed
5. maintainer decision

Approval is not automatic.

## 5. Large files

Do not commit large binary files directly.

If your contribution needs larger data files, describe them first and ask maintainers to publish them as release assets after review.
