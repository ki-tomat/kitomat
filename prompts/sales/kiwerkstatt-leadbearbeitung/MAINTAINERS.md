## Maintenance-Richtlinien

### Issue Triage
- Neue Issues werden innerhalb von 3 Werktagen bewertet
- Labels werden entsprechend vergeben
- Zuständigkeiten werden klar gemacht

### Pull Request Review
- Mindestens 1 Approval nötig vor Merge
- Für größere Änderungen: 2 Reviews erforderlich
- Trust Review bei data_risk ≥ yellow

### Versioning
- Folgt [Semantic Versioning](https://semver.org/)
- CHANGELOG.md wird bei jedem Release aktualisiert
- Tags werden auf GitHub erstellt

### Release-Zyklus
- **Patch-Releases:** Bug-Fixes (beliebig häufig)
- **Minor-Releases:** Neue Features (monatlich)
- **Major-Releases:** Breaking Changes (quartalsweise, geplant)

## Wie man ein Maintainer wird

Kontributos, die konsistent hochwertige Beiträge leisten, können eingeladen werden, Maintainer zu werden. Kriterien:

- ✅ Mindestens 5+ substanzielle PRs
- ✅ Tiefes Verständnis der Codebase/Prompts
- ✅ Gute Kommunikation und Community Spirit
- ✅ Willingness, Code Reviews durchzuführen
- ✅ Einhaltung des Code of Conduct

Bei Interesse: Kontaktiere das Maintainer Team!

## Escalation & Konfliktlösung

Sollte ein Konflikt oder eine Frage entstehen, die nicht sofort gelöst werden kann:

1. Dokumentiere die Frage/Besorgnis im Issue
2. Tagge `@KIWerkstatt-Community-Team`
3. Maintainer werden innerhalb von 5 Werktagen antworten
4. Bei Bedarf wird ein synchrones Meeting arrangiert
