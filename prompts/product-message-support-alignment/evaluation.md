# Evaluation

Dieses Prompt-Paket ist gut nutzbar, wenn die erzeugten Ergebnisse folgende Kriterien erfuellen.

## 1. Konkrete Produktpassung

Das Ergebnis erklaert nicht nur, was das Produkt ist, sondern fuer wen und fuer welchen Einsatzbereich es geeignet ist.

Prueffragen:

- Wird die Zielgruppe klar benannt?
- Werden typische Einsatzsituationen beschrieben?
- Werden Voraussetzungen fuer die Nutzung genannt?
- Wird erklaert, wann das Produkt weniger geeignet ist?

## 2. Keine unbelegten technischen Claims

Das Ergebnis darf keine technischen Eigenschaften, Kompatibilitaeten, Zertifizierungen, Sicherheitsangaben oder Leistungsversprechen erfinden.

Prueffragen:

- Sind technische Aussagen aus den Eingaben ableitbar?
- Werden fehlende Informationen als offene Pruefpunkte markiert?
- Werden unsichere Aussagen vorsichtig formuliert?
- Werden nicht belegte Angaben im Claim-Check sichtbar?

## 3. Vorsichtige Einordnung technischer Angaben

Technische Angaben und Leistungsangaben duerfen nicht ueberinterpretiert werden. Herstellerwerte, Peak-Werte, Maximalangaben oder unvollstaendige technische Angaben muessen vorsichtig formuliert werden.

Prueffragen:

- Werden Leistungsangaben vorsichtig formuliert?
- Werden Peak-, Maximal- oder Herstellerwerte nicht als Garantie dargestellt?
- Werden fehlende Herstellerangaben als offene Pruefpunkte markiert?
- Werden technische Angaben nicht ueberinterpretiert?
- Werden Personenanzahlen, Reichweiten, Outdoor-Eignung oder Kompatibilitaeten nicht pauschal garantiert?

## 4. Grenzen und Nicht-Eignung sind sichtbar

Das Ergebnis nennt nicht nur Vorteile, sondern auch Grenzen.

Prueffragen:

- Gibt es einen Abschnitt "weniger geeignet fuer" oder "nicht geeignet fuer"?
- Werden typische Fehlannahmen angesprochen?
- Werden sicherheitsrelevante Grenzen markiert?
- Werden Ueberversprechen vermieden?

## 5. Supportnutzen

Das Ergebnis soll typische Kundenfragen und Supportfragen vorwegnehmen.

Prueffragen:

- Enthaelt das Ergebnis 5 FAQ-Fragen?
- Enthaelt es 5 Support-Antworten zu haeufigen Kundenfragen?
- Sind die Antworten sachlich und hilfreich?
- Vermeiden die Antworten falsche Sicherheit oder pauschale Freigaben?

## 6. Offizielle Dokumente und Quellenhygiene

Das Ergebnis soll offizielle Dokumente als Pruefpunkte sichtbar machen, ohne Inhalte daraus zu erfinden.

Prueffragen:

- Werden Bedienungsanleitung, Datenblatt, DMX-Chart, Firmware, Software, Kompatibilitaetslisten oder Konformitaetserklaerungen genannt, falls relevant?
- Werden fehlende Dokumente als offene Pruefpunkte markiert?
- Wird die Quellenbasis vorsichtig bewertet?
- Werden Hersteller-Marketingclaims nicht ungeprueft als harte Fakten uebernommen?

## 7. Datenhygiene

Das Ergebnis darf keine echten personenbezogenen oder internen Daten enthalten.

Prueffragen:

- Keine echten Kundennamen?
- Keine echten E-Mail-Adressen?
- Keine kopierten Supporttickets?
- Keine internen Dokumente?
- Keine unklaren Volltextkopien aus Foren oder Reviews?
- Quellenstatus sichtbar?

## 8. Dokumentenaufnahme und Quellenvertrauen

Prueffragen:

- Wurde fuer jede bereitgestellte Datei eine Quellenvertrauensstufe gesetzt?
- Sind offizielle Dokumente von Werbetexten getrennt?
- Werden externe Quellen nur als Hinweis auf Nutzung, Fragen oder Supportfragen verwendet?
- Werden unklare Quellen nicht als Fakten uebernommen?

## 9. Widerspruchspruefung

Prueffragen:

- Werden widerspruechliche Angaben zwischen Dokumenten sichtbar gemacht?
- Werden widerspruechliche Angaben nicht direkt in CMS oder Produktseite uebernommen?
- Gibt es klare offene Pruefpunkte fuer widerspruechliche Daten?

## 10. CMS-/PIM-Output

Prueffragen:

- Gibt es einen strukturierten CMS-/PIM-Feldvorschlag?
- Sind Inhalte je Feld mit Quelle, Vertrauensstufe und Uebernahmestatus versehen?
- Ist klar, welche Inhalte uebernommen, geprueft oder nicht uebernommen werden duerfen?
- Gibt es einen Abschnitt `Nicht veroeffentlichen ohne Pruefung`?
