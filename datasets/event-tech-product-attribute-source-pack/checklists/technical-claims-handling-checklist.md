# Technical Claims Handling Checklist

Dieses Dokument hilft, technische Angaben und Leistungsversprechen vorsichtig zu behandeln, ohne eine zusaetzliche Sonderpruefung einzufuehren.

## Grundsatz

Technische Angaben und Leistungsversprechen duerfen nicht ueberinterpretiert werden.

## Prueffragen

- Stammt die Angabe aus einer offiziellen Quelle?
- Handelt es sich um einen Herstellerwert, Werbeclaim oder externen Erfahrungswert?
- Koennte die Angabe von Nutzern falsch verstanden werden?
- Muss die Aussage vorsichtig formuliert werden?
- Gehoert die Aussage in den Abschnitt `Nicht veroeffentlichen ohne Pruefung`?

## Beispiele

| Angabe | Risiko | Umgang |
|---|---|---|
| Peak-Leistung | wird als Dauerleistung verstanden | klar als Peak kennzeichnen |
| IP-Schutzart | wird falsch als Outdoor-Freigabe interpretiert | nur nach Herstellerangabe nutzen |
| Bluetooth | wird als latenzfrei verstanden | nicht als professionelle latenzfreie Verbindung behaupten |
| Kompatibilitaet | wird zu breit verstanden | konkrete Bedingungen nennen |
